# 2026-03-17

## Fixed swipe indicator and CompletedView visibility issues
- **Issue 1**: Swipe indicator (分隔条) was fixed on screen when PendingView scrolled. Fix: added a scrollable indicator inside PendingView's scroll area and hid the structural one when PendingView is active.
- **Issue 2**: PendingView content (e.g. "3月14日") leaked through when CompletedView was fully visible. Fix: added `visibility: hidden` to PendingView container when `verticalIndex === 0 && verticalDragOffset === 0`.
- Files modified: `src/App.vue`, `src/views/PendingView.vue`

## Swipe physics overhaul + overlay fix
- **Damping model**: Replaced exponential decay damping with hyperbolic hard-limit function (`y = maxPull * x / (x + maxPull * 0.5)`). Max pull distance = 35% of viewport height. Resistance → ∞ at limit.
- **Threshold preserved**: Kept `V_SNAP_THRESHOLD` and velocity-based switching. Only switches view if drag exceeds threshold or fast swipe detected.
- **Dynamic animation duration**: After threshold-triggered switch, animation duration scales with remaining distance (150-300ms).
- **Overlay fix**: Moved swipe overlay from inside individual views to viewport container level. During drag, target view is hidden (`visibility: hidden`) and a full-viewport overlay with bg-primary covers the area. After release, overlay fades out and target view becomes visible.
- **isVerticalDragging**: Upgraded from local `let` variable to `ref` and exposed as `isVerticalDraggingRef` for App.vue consumption.
- Files modified: `src/composables/usePageSwipe.ts`, `src/App.vue`
