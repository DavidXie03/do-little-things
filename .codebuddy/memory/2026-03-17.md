# 2026-03-17

## 过期任务保留 + 已完成任务归档功能

### 修改文件
- `types/index.ts` — 新增 `PastDailyTodos` 接口，`StorageData` 增加 `pastTodos` 和 `completedTodos` 字段
- `composables/storageCore.ts` — 数据版本升级到 6，迁移逻辑添加新字段初始化
- `composables/useDailyTodos.ts` — 新增 `archiveOldTodos()`（日切时归档）、`markPastTodoComplete()`（过期任务完成）、`restoreCompletedTodo()`（已完成恢复为未完成）
- `views/PendingView.vue` — 添加过期任务分组（红色标题+标红文字）、已完成任务可展开列表
- `components/TodoItem.vue` — 新增 `isOverdue` 和 `isCompletedArchive` props
- `i18n/zh.ts` + `i18n/en.ts` — 新增"昨天""前天""已过期""已完成"等翻译

### 设计决策
- 过期未完成任务最多保留 7 天
- 已完成归档最多保留 100 条
- 今天已完成的任务既在"今天"分组中移除，又在"已完成"区域显示
- 已完成区域默认折叠，点击展开
- 过期任务点击红色空心圆可完成（移入已完成区），已完成任务点击绿色圆可恢复未完成

## 已完成任务独立页面（下拉进入）

### 需求
用户在待办列表页面向下拉（pull-down）时进入独立的已完成任务页面。已完成列表只包含**昨天及更早**的已完成任务，今天完成的不计入。

### 新增文件
- `views/CompletedView.vue` — 独立的已完成任务页面，按日期分组展示，支持恢复为未完成

### 修改文件
- `views/PendingView.vue` — 添加下拉触发手势（pull-down indicator）+ 底部"查看已完成历史"按钮入口；移除旧的折叠式已完成区域，仅保留今天已完成的简洁展示
- `App.vue` — 集成 CompletedView 作为从底部滑入的全屏面板，支持打开/关闭动画，Android 返回键支持
- `i18n/zh.ts` + `i18n/en.ts` — 新增 `completed` 翻译组（title, empty, pullDown, release, viewHistory）

### 交互设计
- 在 PendingView 列表滚动到顶部时，向下拉触发，有阻尼效果和视觉提示（箭头图标 + 文字）
- 松手时如果超过阈值则导航到 CompletedView，否则弹回
- CompletedView 从底部滑入，点击返回按钮从底部滑出
- 过滤逻辑：`completedAt < 今天零点` 的任务才会出现在已完成页面

## 垂直页面滑动重构（替换 overlay 面板）

### 需求
用户要求已完成页面和待办列表页的关系像主页和待办列表页一样是连贯的页面切换，而非 overlay 面板。方向为上下滑动（主页⇔待办列表是左右滑动）。标题行"待办列表"在切换时固定不动。切换需有阻力感。

### 修改文件
- `composables/usePageSwipe.ts` — 在现有水平滑动基础上集成垂直滑动系统，共享方向锁定机制。新增：verticalIndex、verticalDragOffset、verticalTranslateY、containerHeight、headerHeight、scrollAreaHeight、goToVerticalPage。垂直阻力参数：阻尼 0.4、阈值 35%、速度阈值 0.5。滚动边界检测：通过 `data-vertical-scroll` 属性查找滚动容器判断 scrollTop/scrollHeight。
- `App.vue` — 移除 completed overlay 面板代码。水平页面1 改为：固定 header（标题+设置按钮） + 垂直滑动区域（CompletedView 上 + PendingView 下，translateY 驱动）。Android 返回键优先处理 verticalIndex。
- `views/PendingView.vue` — 移除 pull-down 手势/指示器/历史入口按钮、header（移到 App.vue）、settings 按钮（移到 App.vue header）。添加 `data-vertical-scroll="pending"`。FAB 可见性增加 verticalIndex 判断。
- `views/CompletedView.vue` — 移除返回按钮和下滑手势。添加 `data-vertical-scroll="completed"`。改用简洁的 section header。
- `i18n/zh.ts` + `i18n/en.ts` — 移除 pullDown/release/viewHistory 翻译。

## 改进：连贯界面 + FAB 修复 + 日期选择

### 需求
1. 删掉待办列表下方独立的"已完成"区块，改为在"今天"分组中混合显示（未完成在前，已完成在后）
2. 下拉时不用抽屉/覆盖面板，而是连贯的上下界面——CompletedView 高度根据内容自适应（不再固定全屏）
3. 修复 FAB 添加按钮与底部 TabBar 重合问题
4. 新增任务时允许选择过去的日期

### 修改文件
- `composables/usePageSwipe.ts` — 改回连贯的上下布局方式（verticalTranslateY 驱动 CompletedView + PendingView 上下滑动），新增 tabBarHeight 变量让 scrollAreaHeight 减去底部 TabBar 高度
- `App.vue` — CompletedView 和 PendingView 纵向排列在同一容器内（flex-col），CompletedView 高度自适应（max-height: 70vh），默认 translateY 隐藏 CompletedView。移除 main 的 padding-bottom，通过 scrollAreaHeight 计算精确高度。用 ResizeObserver 动态跟踪 CompletedView 高度变化
- `views/PendingView.vue` — 恢复 todayCompletedItems，在"今天"分组中合并显示已完成+未完成任务；使用 verticalDragOffset 替代 verticalPanelOffset
- `views/CompletedView.vue` — 恢复为连贯布局的内容自适应高度视图（非覆盖面板）
- `components/TodoModal.vue` — 移除日期选择器的 `:min="getTodayStr()"` 限制，允许选择过去日期

## 三项优化修复

### 需求
1. 取消下拉切换和恢复时的回弹动画
2. FAB 圆形添加按钮出现时增加渐变动画
3. 当新建任务的开始时间是昨天或更早时，要在过期列表中展示遗漏的待办

### 修改文件
- `composables/usePageSwipe.ts` — 修复回弹动画：`animateVerticalTo` 新增 `onComplete` 回调，改用 ease-out quad 缓动；touchEnd 和 goToVerticalPage 改为先动画到目标位置再切换 verticalIndex 并重置 offset，避免跳变回弹
- `views/PendingView.vue` — FAB 按钮用 `<Transition name="fab-fade">` 包裹，添加淡入缩放（opacity + scale）的 CSS 过渡动画
- `composables/useCustomActions.ts` — `addCustomAction` 新增逻辑：当 startDate 在今天之前时，遍历 startDate 到昨天的每个日期，调用 shouldTriggerOnDate 判断，为命中的日期生成 pastTodos 项
