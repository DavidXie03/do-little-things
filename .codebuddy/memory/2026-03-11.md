# 2026-03-11

## 任务循环类型功能 + 待办列表改版

### 新增循环类型（RecurrenceType）
- 新增 5 种循环类型：每天（daily）、每周（weekly）、每月（monthly）、每年（yearly）、工作日（weekday）
- `CustomAction` 和 `Task` 数据模型新增 `recurrence` 字段
- `DailyTodoItem` 新增 `scheduledDate` 字段标记待办属于哪天
- 数据版本从 v4 升级到 v5，包含自动迁移

### 循环触发逻辑（taskService.ts）
- `shouldTriggerOnDate(ca, dateStr)` — 判断某个 CustomAction 在给定日期是否触发
  - weekly/monthly/yearly 以创建日期为锚点
  - weekday 判断 Mon-Fri
- `getNextTriggerDate(ca, fromDateStr)` — 查找下一个触发日（最多 366 天）
- `generateFutureTodoItems()` — 生成未来待办预览列表

### UI 改版
- "今日待办" 改名为 "待办列表"
- TodoModal 新增循环方式选择器（5个按钮横排）
- PendingView 分为"今天"和"即将到来"两个区段
- "即将到来"区段显示每个循环任务的下一个触发日期（每个任务最多一项）
- TodoItem 非每天循环的任务显示循环类型标签
- 默认待办（喝水、活动、眺望）默认设为"每天"循环

### 已完成任务显示下次周期
- 今天已完成的任务保持显示，不消失（撤销之前的隐藏逻辑）
- 已完成任务的副信息行显示下次触发日期，格式如"每周 · 下次 3月18日周三"
- PendingView 中新增 `nextTriggerMap` 计算属性，为每个已完成任务从明天开始查找下一个触发日
- TodoItem 新增 `nextTriggerLabel` prop，`getSubInfo()` 在完成状态时组合"循环类型 · 下次日期"
